# MCQ email setup: Cloudflare + Resend + Gmail

## Target architecture

- Cloudflare Email Routing receives all mail sent to `@mcqpropertycare.com` and forwards it to `mcqpropertycare@gmail.com`.
- Resend sends website email through its API.
- Anthony sends and replies as `anthony@mcqpropertycare.com` from Gmail through Resend SMTP.
- Porkbun is not part of the final mail path.

## Safe migration order

Do not remove Porkbun's MX records until the Cloudflare destination address is verified and the catch-all rule is ready.

### 1. Prepare Cloudflare Email Routing

1. Open **Cloudflare → Compute → Email Service → Email Routing** for `mcqpropertycare.com`.
2. Add `mcqpropertycare@gmail.com` as a destination address.
3. Have Anthony open the Cloudflare verification email in Gmail and approve the destination.
4. Create an explicit route for `anthony@mcqpropertycare.com` to that destination.
5. Enable the catch-all rule and send it to the same destination.
6. Let Cloudflare enable Email Routing and install the root-domain MX records it generates.
7. Remove the old Porkbun MX records and the root SPF record containing `include:_spf.porkbun.com`.
8. Confirm there is only one SPF TXT record at the root. Do not combine Resend's `send` subdomain SPF record with the root record.

Cloudflare DNS continues hosting the website. Only mail-routing MX/TXT records change.

### 2. Verify sending in Resend

1. Add `mcqpropertycare.com` as a sending domain in Resend. Receiving remains disabled.
2. Copy the exact DKIM, SPF/return-path, and verification records generated by Resend into Cloudflare DNS.
3. Keep Resend's MX and SPF records on the hostname Resend specifies, normally `send`; do not replace Cloudflare's root-domain inbound MX records.
4. Wait until Resend reports the domain as verified.
5. Add a DMARC TXT record at `_dmarc.mcqpropertycare.com`, beginning with monitoring policy: `v=DMARC1; p=none; adkim=r; aspf=r`.

### 3. Create separate Resend keys

Create two **Sending access** keys restricted to `mcqpropertycare.com`:

- `MCQ VPS` — stored only in the production `.env` as `RESEND_API_KEY`.
- `MCQ Gmail SMTP` — entered only in Anthony's Gmail SMTP settings.

Do not text, email, commit, or paste either key into chat. Resend shows each key only once.

Production uses:

```env
RESEND_API_KEY=re_...
EMAIL_FROM=Anthony at MCQ <anthony@mcqpropertycare.com>
EMAIL_REPLY_TO=anthony@mcqpropertycare.com
EMAIL_BCC=me@jordangodfrey.com
MERCHANT_SUPPORT_EMAIL=anthony@mcqpropertycare.com
```

`EMAIL_BCC` receives a copy of operational activity mail. The application deliberately suppresses BCC on password resets, verification emails, email-change links, and home invitations because those messages contain private account-access links.

Restart the `handyant` PM2 process after updating the VPS environment.

### 4. Configure Anthony's Gmail

Anthony must do this from Gmail on a computer:

1. Open **Settings → See all settings → Accounts and Import**.
2. Under **Send mail as**, select **Add another email address**.
3. Enter name `Anthony — MCQ Property Care` and address `anthony@mcqpropertycare.com`.
4. Leave **Treat as an alias** enabled.
5. Enter these SMTP settings:

```text
SMTP server: smtp.resend.com
Port: 465
Username: resend
Password: MCQ Gmail SMTP Resend key
Secured connection using SSL: enabled
```

6. Gmail sends a verification message to `anthony@mcqpropertycare.com`. Cloudflare forwards it into the Gmail inbox. Open it and confirm the address.
7. Back under **Send mail as**, make `anthony@mcqpropertycare.com` the default.
8. Edit that address and set its reply-to address to `anthony@mcqpropertycare.com`.
9. For **When replying to a message**, choose **Reply from the same address the message was sent to**.

## End-to-end verification

Use an outside email account, not `mcqpropertycare@gmail.com`, for the tests.

1. Email `anthony@mcqpropertycare.com`; confirm it arrives in Gmail.
2. Email a made-up address such as `routing-test@mcqpropertycare.com`; confirm the catch-all forwards it.
3. Reply from Gmail; confirm the recipient sees `Anthony — MCQ Property Care <anthony@mcqpropertycare.com>` and not the Gmail address.
4. Send a new message from Gmail using the From selector and confirm the same result.
5. Create a controlled customer invitation from the MCQ website and confirm the app email arrives, its links work, and replies go to Anthony.
6. Check the Resend log for delivered status and verify no unexpected bounces.

Do not consider the migration complete until all six checks pass.
